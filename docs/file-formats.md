# Supported File Formats

Genetica Resolutio reads raw genotype export files from all major consumer DNA providers. It also reads standard VCF files. Gzip-compressed files are supported natively — you do not need to decompress them before uploading.

---

## Supported providers at a glance

| Provider | File type | Extension | Notes |
|---|---|---|---|
| 23andMe | Tab-separated text | `.txt` or `.txt.gz` | All chip versions (v1–v5+) |
| AncestryDNA | Tab-separated text | `.txt` or `.txt.gz` | All versions |
| MyHeritage | Comma-separated text | `.csv` or `.csv.gz` | — |
| FamilyTreeDNA | Tab-separated text | `.csv` or `.txt` | — |
| LivingDNA | Tab-separated text | `.txt` | — |
| Generic VCF | Variant Call Format | `.vcf` or `.vcf.gz` | Standard spec; see VCF section below |

---

## Format details

### 23andMe

23andMe raw data files are tab-separated text files. Comment lines at the top begin with `#` and typically include a line identifying the source (`# This data file generated by 23andMe`).

**Column structure (4 columns):**
```
rsid    chromosome    position    genotype
rs4477212    1    82154    AA
rs3094315    1    752566    AG
```

The `genotype` column is a two-character string (e.g., `AA`, `AG`, `--`). Insertions/deletions (`I`, `D`) and no-calls (`--`) are skipped.

### AncestryDNA

AncestryDNA files are tab-separated with a header line and 5 columns. Comments begin with `#` and include `# AncestryDNA`.

**Column structure (5 columns):**
```
rsid    chromosome    position    allele1    allele2
rs4477212    1    82154    A    A
rs3094315    1    752566    A    G
```

The two alleles are in separate columns. The parser combines them into the standard `A1/A2` format used throughout the app.

### MyHeritage

MyHeritage files are comma-separated with a header line. Comments begin with `#` and include `MyHeritage`.

**Column structure (4 columns, comma-separated):**
```
RSID,CHROMOSOME,POSITION,RESULT
rs4477212,1,82154,AA
rs3094315,1,752566,AG
```

The `RESULT` column is a two-character string identical to 23andMe's `genotype` column.

### FamilyTreeDNA

FamilyTreeDNA autosomal raw data files are tab-separated. The format is similar to AncestryDNA with 5 columns. There may be no provider-identifying comment, so the parser falls back to column count detection.

### LivingDNA

LivingDNA files use a tab-separated format similar to 23andMe. Provider identification is from comment lines.

### VCF (Variant Call Format)

Standard VCF files (version 4.x) are supported. This covers output from sequencing labs, clinical genomics pipelines, and some third-party data analysis tools.

**Requirements for VCF files:**
- rsIDs must be present in the `ID` column (column 3)
- The `REF`, `ALT`, and `FORMAT`/`SAMPLE` columns are used to reconstruct the genotype
- Multi-sample VCF files: only the first sample column is read
- Structural variants and complex alleles are skipped — only single-nucleotide variants are analyzed
- VCF files may be gzip-compressed (`.vcf.gz`)

---

## Compression

Files compressed with gzip are read directly without needing to decompress them first. The app detects gzip format by inspecting the first two bytes of the file (magic bytes `0x1f 0x8b`) rather than relying on file extension.

This means:
- `genome.txt.gz` — works
- `genome.txt` — works
- `genome.gz` — works (regardless of extension)
- `.zip` archives — **not** supported; extract the file from the archive first

If your provider delivers a `.zip` file, open it with your OS archive manager and extract the inner `.txt` or `.csv` file before uploading.

---

## What gets parsed

For each line of the input file, the parser extracts:

1. **rsID** — must match the pattern `rs` followed by digits (e.g., `rs1801133`). Lines without a valid rsID are skipped.
2. **Alleles** — must consist only of the characters `A`, `T`, `C`, `G`. Lines with insertion/deletion markers (`I`, `D`), no-call markers (`--`, `0`), or invalid characters are skipped.

Lines that begin with `#` are treated as comments and skipped regardless of content.

## File size expectations

Consumer genotyping arrays typically cover 500,000 to 900,000 SNPs. This corresponds to raw data files of roughly:

| Provider | Compressed size | Uncompressed size |
|---|---|---|
| 23andMe (v5+) | 5–8 MB | 25–35 MB |
| AncestryDNA | 3–6 MB | 15–25 MB |
| MyHeritage | 5–10 MB | 20–40 MB |
| FamilyTreeDNA | 3–7 MB | 15–30 MB |

Analysis time is 1–5 seconds for all of these. Very large files (e.g., whole-genome sequencing data in VCF format, potentially hundreds of MB to several GB) will take longer but are fully supported as long as they follow the VCF specification.

---

## Format detection

The app detects file format automatically. It does not rely on the file extension — it reads the content to determine the format. Detection order:

1. Check for gzip magic bytes → decompress if present, then continue
2. Read the first non-comment line
3. Count delimiters (tab vs. comma) to distinguish tab-separated from CSV
4. Count columns: 5 columns → AncestryDNA/FTDNA format; 4 columns → 23andMe/MyHeritage format
5. Check whether the first column header is `RSID` (uppercase) vs `rsid` (lowercase) for additional provider disambiguation
6. If the first non-comment line starts with `##` followed by `fileformat=VCF` → VCF mode

If format detection fails or produces unexpected results, check the [FAQ](faq.md) for troubleshooting steps.

---

## Providers not listed here

If your DNA provider is not in the list above, the file may still work if it follows a compatible format. Try uploading it — the parser is tolerant of minor variations in header names and column ordering as long as an rsID and genotype can be identified.

If the analysis produces a very low matched count (under 20 variants), the file likely wasn't parsed correctly. See the [FAQ](faq.md) for what to check.
